# LCM Stats Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a read-only `lcm_stats` observability tool and CLI command that report aggregate LCM index shape without exposing raw transcript text.

**Architecture:** Implement a storage-level stats method that reuses the existing SQLite index when available and returns stable aggregate counts. Expose that method through the MCP server as `lcm_stats`, plus a matching `codex-lcm stats --json` CLI command for local diagnostics.

**Tech Stack:** TypeScript, Node.js, built-in `node:sqlite`, MCP JSON-RPC test harness, existing `node --test` suite.

---
