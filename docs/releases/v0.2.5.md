# Codex LCM v0.2.5

`v0.2.5` recovers context within the turn that triggered compaction and reduces
unnecessary storage work.

## Fixes

- ([#38](https://github.com/Team-Volt/codex-lcm/pull/38)) Stop the `PostCompact` hook from returning unsupported output while preserving its pending recovery marker.
- ([#39](https://github.com/Team-Volt/codex-lcm/pull/39)) Inject the recovery directive after the next tool result and block turn completion until `lcm_pack_context` clears the pending marker.
- ([#40](https://github.com/Team-Volt/codex-lcm/pull/40)) Reconcile edited or truncated raw logs, reject malformed optional string arrays at the MCP boundary, preserve unchanged summary rows during rebuilds, and query top summary nodes directly.

## Upgrade to v0.2.5

Restart Codex Desktop or open a fresh Codex CLI/TUI session after upgrading so
the refreshed plugin cache, MCP server, hooks, and skill are loaded.

For an existing GitHub marketplace install:

```sh
codex plugin marketplace remove codex-lcm
codex plugin marketplace add Team-Volt/codex-lcm --ref v0.2.5
codex plugin add codex-lcm@codex-lcm
```

For a local checkout install:

```sh
git -C /path/to/codex-lcm fetch --tags origin
git -C /path/to/codex-lcm checkout v0.2.5
codex plugin add codex-lcm@codex-lcm
```
