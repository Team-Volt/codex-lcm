# Codex LCM v0.2.0

`v0.2.0` is the first tagged Codex LCM developer release. It packages the
current local-first memory workflow for Codex: native plugin installation,
automatic hook capture, sanitized raw event storage, local search, DAG-backed
retrieval, summary-node expansion, diagnostics, and transcript import.

## Features

- ([#18](https://github.com/Team-Volt/codex-lcm/pull/18)) Add `doctor` and `import-codex-sessions` commands for install diagnostics and backfilling existing Codex session JSONL files.
- ([#17](https://github.com/Team-Volt/codex-lcm/pull/17)) Add query-first recursive evidence expansion with `lcm_expand_query`.
- ([#16](https://github.com/Team-Volt/codex-lcm/pull/16)) Capture `PostCompact` hook events so LCM can tell when compaction completed.
- ([#15](https://github.com/Team-Volt/codex-lcm/pull/15)) Add standard LCM MCP verbs: `lcm_grep`, `lcm_describe`, `lcm_expand`, and `lcm_expand_query`.
- ([#12](https://github.com/Team-Volt/codex-lcm/pull/12)) Expose hook event counts in stats output.
- ([#10](https://github.com/Team-Volt/codex-lcm/pull/10)) Add LCM stats observability for graph, summary, freshness, and hook-capture checks.
- ([#3](https://github.com/Team-Volt/codex-lcm/pull/3)) Add deterministic extractive session summaries.
- ([#2](https://github.com/Team-Volt/codex-lcm/pull/2)) Improve broad LCM context retrieval with packed context support.

## Upgrade to v0.2.0

Restart Codex Desktop or open a fresh Codex CLI/TUI session after upgrading so
the refreshed plugin cache, MCP server, hooks, and skill are loaded.

For an existing GitHub marketplace install:

```sh
codex plugin marketplace remove codex-lcm
codex plugin marketplace add Team-Volt/codex-lcm --ref v0.2.0
codex plugin add codex-lcm@codex-lcm
```

If `codex plugin marketplace remove codex-lcm` says the marketplace does not
exist, continue with the `marketplace add` command.

For a local checkout install:

```sh
git -C /path/to/codex-lcm fetch --tags origin
git -C /path/to/codex-lcm checkout v0.2.0
codex plugin add codex-lcm@codex-lcm
```
