# Codex LCM v0.2.0

`v0.2.0` is the first tagged Codex LCM developer release. It packages the
current local-first memory workflow for Codex: native plugin installation,
automatic hook capture, sanitized raw event storage, local search, DAG-backed
retrieval, summary-node expansion, diagnostics, and transcript import.

## Highlights

- Native Codex plugin packaging with MCP server, lifecycle hooks, and the
  `lcm-recall` skill.
- Sanitized append-only event capture with local JSONL storage before indexing.
- SQLite FTS plus a derived DAG of sessions, turns, events, tool results,
  checkpoints, and summary-source lineage.
- Deterministic extractive summaries and recursive summary-node retrieval for
  source-backed recall without loading full transcripts.
- MCP tools for health, stats, current-session lookup, grep, describe, expand,
  query expansion, context packing, graph inspection, raw-event paging, and note
  recording.
- Diagnostics for install wiring, storage health, capture state, hook counts,
  summary indexes, and Codex session import readiness.
- `import-codex-sessions` support for backfilling existing Codex transcript
  JSONL files without modifying source transcripts.
- Local-only operation with no runtime npm dependencies, hosted services, or
  embeddings required.

## Upgrade

Restart Codex Desktop or open a fresh Codex CLI/TUI session after upgrading so
the refreshed plugin cache, MCP server, hooks, and skill are loaded.

For an existing GitHub marketplace install:

```sh
codex plugin marketplace remove codex-lcm
codex plugin marketplace add Team-Volt/codex-lcm --ref v0.2.0
codex plugin add codex-lcm@codex-lcm
```

If `codex plugin marketplace remove codex-lcm` says the marketplace does not
exist, continue with the `marketplace add` command.

For a local checkout install:

```sh
git -C /path/to/codex-lcm fetch --tags origin
git -C /path/to/codex-lcm checkout v0.2.0
codex plugin add codex-lcm@codex-lcm
```
